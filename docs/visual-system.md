# AI Today Visual System

AI Today should feel like a public-interest briefing desk: calm, sourced, Canadian, and useful under time pressure. The system should avoid generic AI imagery and lean on editorial signals: rules, ledgers, maps, issue numbers, source marks, and restrained data graphics.

## Core Idea

**Signal desk.** Each page should feel like part of a daily intelligence file: a clear date, a clear issue state, source-linked evidence, and a small amount of visual signal.

## Visual Rules

- Use the brand name once per surface when possible. Let dates, issue numbers, and sections carry local context.
- Prefer ruled lists and metadata ledgers over card-heavy layouts.
- Use one accent: maple red `#B54818`.
- Keep backgrounds warm paper, not bright white or dark SaaS.
- Use serif display type for editorial hierarchy and sans type for labels, metadata, controls, and admin surfaces.
- Keep graphics quiet: maps, documents, data marks, signal traces. Avoid robots, glowing brains, chat bubbles, purple gradients, and fake UI screens.

## Reusable Patterns

- **Signal ledger:** a compact metadata strip for issue state, publishing rhythm, and editorial standard.
- **Issue row:** issue number plus clean date title; no repeated brand/date pairs.
- **Source standard:** source-linked, AI-assisted, Canada-first appears as a system note, not repeated prose.
- **Issue graphic:** paper/map/report visuals, used sparingly at hero scale.
- **Section treatment:** section name, short description, count, and a subtle data mark.

## Page Guidance

- Homepage: introduce the publication once, show the latest published issue, then archive entry points.
- Issues archive: chronological record, date-led rows, minimal explanation.
- Issue page: date-led headline can keep the publication title in metadata/SEO, but visible reading flow should prioritize issue date, sections, stories, and sources.
- Admin: operational and denser, using the same type and colour tokens but less visual decoration.
